Customer Problems and Solutions:

In the manufacturing industry, our clients faced a multitude of challenges. Production inefficiencies, lack of real-time data, and complex shift coordination were common problems. To address these issues, we designed the Sonyu Plastic MES Industry Automation System. This solution empowers manufacturers to achieve efficiency and control, make data-driven decisions, and navigate the complexities of multi-shift operations.

Technical Details:

The Sonyu Plastic MES Industry Automation System leverages a robust technology stack to deliver its powerful functionality:

  • Framework: Built on the Laravel PHP framework, ensuring scalability and maintainability.
  • Machine Connectivity: Utilizes MQTT for efficient and secure communication with manufacturing machines.
  • API Integration: Seamlessly integrates with various APIs for data exchange and interoperability with existing systems.
  • Node-RED: Employs Node-RED for connecting machines to Programmable Logic Controllers (PLCs), ensuring seamless automation and control.

Challenges and Solutions:

Throughout this project, our team encountered various challenges, such as addressing unplanned downtime and creating efficient maintenance schedules. Our problem-solving skills came to the forefront as we developed solutions that minimized disruptions and streamlined complex order management.

  • Timezone Management: Managing production status across different time zones posed a challenge. To auto-end production status on day changes and handle shifts effectively while considering daylight saving, we adopted a solution by working with UTC + 0 and adding a static +1 offset. This approach enabled us to accurately run shifts and collect data based on a 24-hour cycle.
  • Data Discrepancies: We encountered issues with missing machine bottle counts in the company's data. To address this, we implemented a trigger system. It verified the time of machine shutdown and network activity, cross-referencing this with data sent by Node-RED. By matching manufacturing order times with machine production times, we successfully reconstructed the missing data, ensuring data integrity.
  • API Data Overload: Dealing with multiple threads and rapid data influx from APIs created multiple data rows in the database within seconds. To streamline data for reporting, we implemented preprocessing techniques. This allowed us to separate and manage data efficiently, ensuring accurate and meaningful reports.
